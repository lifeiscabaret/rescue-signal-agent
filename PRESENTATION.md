## 2. 발표 스크립트

### 🟠 오프닝 (0:00 ~ 0:45)

> **"펫샵에 가기 전, 지금 이 순간에도 공고 마감이 다가오는 아이들이 있습니다."**

안녕하세요, 팀 [팀명]입니다.

저희가 해결하고자 한 문제는 단순한 정보 탐색 문제가 아닙니다.
**행동 전환 문제**입니다.

유기동물 정보는 이미 공공데이터, SNS, 보호소 공고 어디에나 있습니다.
하지만 정작 도움이 필요한 아이를 찾고, 문의하고, 행동으로 이어지기까지
많은 사람들이 중간에 포기합니다.

마치 청년주택 알림처럼 — 조건에 맞는 공고가 뜨면 바로 알림이 오는 것처럼 —
**구조동물도 그렇게 될 수 있다면 어떨까요?**

---

### 🟡 솔루션 소개 (0:45 ~ 1:30)

Rescue Signal Agent는 5개의 AI 에이전트가 협력합니다.

```
사용자가 조건을 입력하면
→ 에이전트가 공공데이터를 조회하고
→ 공고 마감일과 특이사항으로 우선순위를 판단하고
→ 사용자 조건과 매칭하고
→ 보호소 문의 문구와 SNS 공유 문구까지 생성하고
→ Discord로 알림을 보냅니다
```

단순한 목록 조회가 아닙니다.
에이전트가 **판단하고, 매칭하고, 행동을 돕습니다.**

---

### 🟢 라이브 데모 (1:30 ~ 3:30)

**[지현 데모 진행]**

#### 데모 시나리오

```
입력:
"저는 서울에 살고, 강아지 임시보호가 가능해요.
소형견이면 좋지만 노령견도 괜찮고,
직접 데려오기는 어렵지만 SNS 홍보는 할 수 있어요."
```

#### 데모 포인트 (지현이 화면을 보여주면서)

1. **사용자 조건 입력** → JSON 또는 자연어
2. **에이전트 실행** → 로그로 5개 에이전트 순서 보여주기
3. **TOP 3 출력**:
   - 각 아이의 `priorityScore` / `matchScore` 표시
   - 추천 이유 한 줄 설명
4. **문구 생성**:
   - 보호소 문의 문구 출력
   - SNS 공유 문구 출력
5. **Discord 알림** → 실제 Discord 채널에 메시지 전송

#### 데모 멘트 (경윤)

> "보시는 것처럼 에이전트가 공고 마감이 가까운 순서로 우선순위를 정하고,
> 사용자가 임보 가능하고 소형견을 원한다는 조건과 매칭합니다.
> 그리고 직접 보호소에 전화하기 어려운 분들을 위해 문의 문구까지 생성합니다.
> 마지막으로 Discord 채널로 알림이 전송됩니다."

#### ⚠️ 데모 안전망

| 상황 | 대응 |
|------|------|
| 공공데이터 API 응답 없음 | sample_animals.json 자동 fallback |
| Azure 연결 오류 | 로컬 모드로 전환, 결과 동일 |
| Discord 전송 실패 | 메시지 텍스트 화면 표시로 대체 |

### 🔵 기술 아키텍처 (3:30 ~ 4:15)

**[슬라이드 또는 README 아키텍처 다이어그램]**

저희는 전체 스택을 Microsoft 기술로 구성했습니다.

| 레이어 | Microsoft 기술 |
|--------|--------------|
| AI 에이전트 핵심 | **Microsoft Agent Framework (MAF)** |
| LLM | **Azure AI Foundry / GPT-4o** |
| RAG | **Azure AI Search** — 안전 표현 정책 grounding |
| 알림 도구 | **MCP-ready Notification Tool** → Discord |
| 서버리스 스케줄러 | **Azure Functions** |
| 배포 | **Azure Container Apps** |
| 코드 생성 전반 | **GitHub Copilot** |

특히 **Azure AI Search RAG**를 통해
에이전트가 "안락사 예정" 같은 단정 표현 대신
"우선 확인 필요"라는 안전한 표현을 사용하도록 grounding했습니다.

### 🔴 비즈니스 임팩트 & 마무리 (4:15 ~ 5:00)

**숫자로 보는 임팩트**

- 연간 유기동물 113,000마리 중 입양률 현재 32%
- 저희 서비스로 **45%까지 향상** 가능 추산 → 연간 14,000마리 추가 입양
- 보호소 홍보 업무 **90% 자동화**

**확장 가능성**

이 에이전트 패턴은 유기동물만이 아닙니다.
청년주택 공고, 취업 공고, 재난 정보 —
모든 공공데이터 기반 개인화 알림 서비스에 적용 가능한
**재사용 가능한 에이전트 템플릿**입니다.

**마무리 문장**

> "펫샵에서 구매하기 전, 구조가 필요한 아이들을 먼저 확인할 수 있게 —

## 4. 예상 Q&A

**Q. 공공데이터 API가 불안정하면 어떻게 되나요?**
> A. sample data fallback이 자동으로 작동합니다. 데모 안정성이 보장됩니다.

**Q. 실제 안락사 여부를 판단할 수 있나요?**
> A. 공공데이터만으로 안락사 여부를 단정할 수 없습니다. 그래서 저희는 "우선 확인 필요"라는 표현을 사용하며, RAG를 통해 에이전트가 안전한 표현만 사용하도록 설계했습니다.

**Q. 왜 Discord를 선택했나요?**
> A. MVP에서는 가장 빠르게 알림 데모를 보여줄 수 있기 때문입니다. MCP-ready 구조로 설계했기 때문에 Teams, Slack, KakaoTalk으로 확장이 용이합니다.

**Q. 비즈니스 모델은 무엇인가요?**
> A. 보호소/지자체 B2B SaaS, 동물보호단체 파트너십, 이후 Microsoft Azure Marketplace 등록 목표입니다.
