# 📋 PRD — Product Requirements Document

> **심사 기준 4: Business Impact & Viability (목표: 20/20)**

---

## 1. 제품 비전

**"한 마리도 놓치지 않는 구조 신호"**

Rescue Signal Agent는 분산된 유기동물 데이터를 AI 에이전트가 통합·분석하여,  
입양·임보가 가장 급한 동물에게 **자동으로 구조 신호를 보내는 서비스**입니다.

> 💡 **핵심 포지셔닝**: 펫샵에서 구매하기 전, 구조가 필요한 아이들을 먼저 확인할 수 있게 돕는 공공데이터 기반 입양 전환 AI 에이전트

---

## 2. 제품 목표

| 목표 | 설명 | KPI |
|------|------|-----|
| **데이터 통합** | 공공데이터 API 기반 실시간 수집 | 수집 지연 < 1시간 |
| **긴급도 자동화** | AI 스코어링으로 수동 작업 제거 | 일 100건+ 자동 처리 |
| **알림 정확도** | 관련성 높은 사용자에게만 발송 | 알림 오탐률 < 15% |
| **홍보 콘텐츠 생성** | 보호소용 SNS 초안 자동 생성 | 콘텐츠 생성 시간 90% 단축 |

## 3. 사용자 스토리 (User Stories)

### 🐶 입양 희망자
```
AS A 입양을 고려 중인 시민
I WANT 내 조건(종, 지역, 나이)에 맞는 동물이 등록되면 알림을 받고 싶다
SO THAT 좋은 타이밍을 놓치지 않고 입양 결정을 할 수 있다
```

### 🏠 임보 봉사자
```
AS A 단기 임보를 제공할 수 있는 봉사자
I WANT 내 지역의 긴급 임보 요청을 자동으로 받고 싶다
SO THAT 직접 보호소에 전화하지 않아도 급한 아이들을 도울 수 있다
```

### 🏥 보호소 담당자
```
AS A 지방자치단체 동물보호센터 직원
I WANT 보호 중인 동물의 SNS 홍보 콘텐츠를 자동으로 받고 싶다
SO THAT 수작업 없이 입양 홍보 효과를 높일 수 있다
```

## 5. 비기능 요구사항 (Non-Functional Requirements)

| 항목 | 요구사항 |
|------|---------|
| **가용성** | 99.5% 이상 (Azure App Service) |
| **응답 시간** | 스코어링 API < 2초 |
| **보안** | API 키 Azure Key Vault 관리, PII 최소 수집 |
| **확장성** | 전국 500+ 보호소 데이터 처리 가능 |
| **컴플라이언스** | 개인정보보호법 준수, 공공데이터 이용조건 준수 |

## 6. 비즈니스 임팩트 & ROI

### 사회적 가치
| 지표 | 예상 효과 |
|------|---------|
| 입양률 향상 | 현재 32% → 45% (연간 14,000마리 추가 입양) |
| 안락사 감소 | 연간 3,000~5,000마리 감소 |
| 보호소 업무 절감 | 홍보 업무 주 10시간 → 1시간 이하 |

### 경제적 가치
| 항목 | 금액 |
|------|------|
| 1마리 보호 비용 (평균 10일) | 약 15만원 |
| 연간 안락사 감소 3,000마리 기준 절감 | 약 **45억원** |
| 보호소 담당자 시간 절감 (500개소 × 9h/week) | 연간 234,000시간 |
