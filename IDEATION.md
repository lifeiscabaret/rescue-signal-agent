# 💡 IDEATION — 문제 정의 & 아이디어

## 1. 문제 인식 (Problem Statement)

### 핵심 문제
> "유기동물 정보는 공공데이터와 SNS에 흩어져 있고, 하루만 지나도 중요한 공고가 묻힌다."

Rescue Signal Agent는 이 문제를 단순한 **"정보 탐색 문제"**가 아니라  
**"행동 전환 문제"**로 정의합니다.

유기동물 정보는 이미 존재합니다. 하지만 실제로 도움이 필요한 아이를 찾고,  
문의하고, 입양·임보 행동으로 이어지기까지의 과정이 너무 어렵습니다.

> 💡 **핵심 메시지**: 펫샵에서 구매하기 전, 구조가 필요한 아이들을 먼저 확인할 수 있게 돕는 공공데이터 기반 입양 전환 AI 에이전트

전국 보호소에서 **매일 수백 건**의 유기동물 공고가 등록·수정·종료됩니다.  
하지만 현재 시스템에는 구조적 한계가 있습니다:

| # | 문제 | 현황 |
|---|------|------|
| 1 | **긴급한 아이들이 정보 흐름에 묻힘** | 하루만 지나도 새로운 공고와 SNS 게시물에 밀려 중요한 정보가 지나감 |
| 2 | **사용자가 직접 판단해야 할 요소가 많음** | 지역, 보호소, 공고 기간, 동물 상태, 임보 가능성, 문의 방법을 직접 비교해야 함 |
| 3 | **입양보다 펫샵 구매가 더 쉽게 느껴짐** | 펫샵은 원하는 조건의 동물을 빠르게 보여주지만, 구조동물 입양은 탐색과 문의 과정이 어려움 |
| 4 | **알림 구조가 부족함** | 청년주택 알림처럼 조건에 맞는 정보를 받아보는 구조가 없어, 직접 찾아보지 않으면 놓침 |

## 2. 시장 규모 & 사회적 임팩트

### 국내 유기동물 현황 (2024 기준)
- **연간 유기·유실동물 입소**: 약 **113,000마리** (농림축산식품부)
- **안락사율**: 약 **10~15%** → 연간 10,000~17,000마리
- **자연사 포함 총 사망**: 연간 약 27,000마리
- **입양률**: 약 **32%** (개선 여지 큼)

### Why Now?
- 2023년부터 공공데이터 API 무료 개방 (농림축산식품부)
- Microsoft Azure AI + MAF로 LLM 기반 에이전트 구축 비용 급감
- 반려동물 인구 **1,500만 명** 시대 → 사회적 관심 최고조

## 3. 타겟 고객 & Pain Point

### Persona 1: 입양 희망자 (일반 시민)
- **Pain**: "마음에 드는 강아지가 있었는데, 다음날 확인하니 이미 공고 종료"
- **Need**: 관심 동물 등록 시 상태 변경 즉시 알림

### Persona 2: 임보 봉사자
- **Pain**: "임보가 급한 아이들을 직접 보호소 전화로 확인해야 함"
- **Need**: 긴급 임보 요청 자동 필터링 & 알림

### Persona 3: SNS 홍보자
- **Pain**: "직접 입양/임보는 어렵지만, 도움이 필요한 아이를 빠르게 공유하고 싶다"
- **Need**: 홍보 문구·해시태그 자동 생성, 공유 한 번으로 도움 가능

### Persona 4: 보호소/단체 운영자
- **Pain**: "아이들의 공고가 묻히지 않도록 홍보 문구와 알림 채널을 자동화하고 싶다"
- **Need**: 홍보 콘텐츠 자동 생성 + Discord/메신저 알림 자동화

## 4. 기존 솔루션의 한계

| 서비스 | 한계점 |
|--------|--------|
| 포인핸드 앱 | 검색 기반, 긴급도 자동 분류 없음 |
| 농림축산식품부 공고 | 단순 리스트, UI 불편, 알림 없음 |
| 카라/동물자유연대 SNS | 수동 업로드, 체계적 우선순위 없음 |
| 지자체 홈페이지 | 각 지자체마다 별도 접속 필요 |

→ **AI 기반 자동화 + 우선순위 스코어링 + 통합 알림** 솔루션은 현재 시장에 없음

## 5. 해결 방향 (Solution Direction)

```
[수집] 공공데이터 API 주기 폴링
    ↓
[분석] LLM 기반 긴급도 스코어링 (D-day, 품종, 지역, 상태)
    ↓
[매칭] 사용자 프로필 기반 추천 (임보/입양/홍보 구분)
    ↓
[알림] 채널별 맞춤 발송 (앱 푸시, 이메일, 카카오톡)
    ↓
[리포트] 주간 보호소별 현황 자동 요약
```

### 핵심 차별점: **긴급도 스코어 알고리즘**
```
긴급도 점수 = 
  (보호 종료까지 남은 일수 × -3)
  + (입양 공고 후 경과일 × 1)
  + (품종 희귀도 보정)
  + (건강 상태 가중치)
  + (재보호 이력 × 2)
```

## 6. MVP 범위 (Hackathon 데모)

### In Scope ✅
- 공공데이터 API 연동 (농림축산식품부 유기동물 조회)
- 긴급도 자동 스코어링
- 상위 N개 긴급 동물 리스트 출력
- 간단한 알림 메시지 자동 생성

### Out of Scope ❌ (v2 목표)
- 실사용자 앱 배포
- SNS 자동 포스팅
- 보호소 담당자 전용 대시보드
